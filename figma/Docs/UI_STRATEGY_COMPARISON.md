# UI 구현 전략 비교

 React 프로토타입을 LWC로 변환할 때의 **3가지 전략** 비교

## 전략 요약

| 전략 | Custom CSS | SLDS | 원본 일치도 | 권장 상황 |
|------|-----------|------|-----------|----------|
| **A. 픽셀 퍼펙트** | 90% | 10% | ⭐⭐⭐⭐⭐ | 디자인 정확성 최우선 |
| **B. 하이브리드** | 60% | 40% | ⭐⭐⭐⭐ | **✅ 대부분의 경우 권장** |
| **C. SLDS 표준** | 20% | 80% | ⭐⭐ | Salesforce 표준 최우선 |

## 권장: 전략 B (하이브리드)

### Custom CSS 영역 (60%)
- 2-패널 레이아웃
- 온톨로지 트리 스타일
- 상태 뱃지
- 체크리스트 배너
- 색상 시스템 전체

### SLDS 활용 영역 (40%)
- `<lightning-input>` - 모든 입력 필드
- `<lightning-button>` - 모든 버튼
- `<lightning-modal>` - 모달 기본 구조
- `<lightning-icon>` - 아이콘

**예상 결과**: 디자인 일치도 92-95%
